JS继承方式

1.原型链方式

分析：

Child是如何获取name的

  1. 首先在Child对象自身中找。若对象自身没找到
  2. 然后在Child.prototype中找。若Child.prototype中没找到
  3. 继续往上一层，Child.prototype.proto(Parent.prototype)
  4. 依次类推，直到找到需要的属性或方法，或到达原型链顶端Object.prototype

缺点：原型链中引用类型的属性会被所有实例共享的，即所有实例对象使用的是同一份数据，会相互影响。

2.构造函数方式

分析：

这种方法的思想就是在子类构造函数的内部调用父类构造函数，可以借助apply()和call()方法来改变对象的执行上下文

缺点：方法都在构造函数中定义，因此函数无法达到复用

3.组合继承（原型链+构造函数）

分析：

组合继承是将原型链继承和构造函数结合起来，从而发挥二者之长的一种模式。

思路就是使用原型链实现对原型属性和方法的继承，而通过借用构造函数来实现对实例属性的继承。

这样，既通过在原型上定义方法实现了函数复用，又能够保证每个实例都有它自己的属性。
